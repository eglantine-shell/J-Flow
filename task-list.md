# 开发任务清单

本文档用于指导 V2 后续工作拆分。

说明：
- V1 已完成可试用 MVP，相关实现记录保留
- 当前任务清单改为面向 V2 的迁移与重构阶段
- 本轮只重写任务拆分，不执行实现

---

## V2 总目标

围绕以下方向推进：
- Todo 成为主页面主体
- 种草成为独立清单
- 拔草推荐成为“有空时”的增强能力
- 尽量复用现有数据结构与既有能力
- 只在后续实现中确认必要时，再调整数据模型

---

## V2-1：文档与概念迁移

### 目标
统一产品叙事，把 V1 的“决策库 / 决策推荐 / 双模式”迁移为 V2 的“Todo / 种草 / 拔草”。

### 要求
- 明确 Todo 是主页面主体
- 明确种草不再承担普通待办职责
- 明确拔草推荐是增强能力，而非唯一入口
- 保留 V1 历史记录，但明确当前方向已转向 V2

### 交付物
- `README.md` 更新
- `product-rules.md` 更新
- `app-structure.md` 更新
- `handoff.md` 更新
- `dev-log.md` 记录本轮迁移背景和判断

---

## V2-2：初始化页文案与默认项调整

### 目标
把初始化页从“时间场景 / 活动类型设置”迁移为更符合 V2 心智的引导页。

### 要求
- 标题调整为：
  - 哪怕你是一个100%的J人，想必也会有……
- 第一组改为“种草清单”
- 第二组改为“有空就做”
- 默认分类调整为：
  - 阅读
  - 观影
  - 旅游
- 默认场景调整为：
  - 工作日晚上
  - 周末
  - 长假
- 底层数据结构仍复用 `activityTypes` 与 `sceneTags`

### 交付物
- 初始化页文案方案
- 默认 seed 调整方案
- 与现有字段映射关系确认

---

## V2-3：主页面从“双模式”改为“Todo + 种草”单页结构

### 目标
将主页面从 V1 的“决策模式 / Todo 模式”双模式，迁移为 V2 的单页结构。

### 要求
- 主页面结构改为：
  1. 头部
  2. 日期
  3. Todo
  4. 种草
- Todo 成为视觉和交互主体
- 种草成为独立区域
- 决策模式相关表达迁移为拔草能力，而不是独立模式

### 交付物
- 页面信息结构方案
- 模块迁移清单
- 旧结构到新结构的映射说明

---

## V2-4：Todo 添加入口融合临时事项与种草推荐

### 目标
让 Todo 新增入口同时支持“先快速记事”和“按需拔草”。

### 要求
- 默认新增为临时事项
- 添加入口中融合：
  - 快速新增临时事项
  - 从种草中选择
  - 推荐拔草
- 允许展开更多选项：
  - 必要事项
  - 重复规则
  - 需要准备
  - 分次事项

### 交付物
- 添加入口交互方案
- 临时事项与加入安排的融合方案
- 与现有字段能力的映射说明

---

## V2-5：Todo List UI 轻量化

### 目标
让 Todo 回到真正轻量、顺手、低门槛的使用体验。

### 要求
- 弱化模板感
- 弱化重表单感
- 强化“快速记下今天要做的事”
- 保留必要的准备、进度、时段语境等信息，但不压过主任务本身
- 继续保留白天 / 晚上语境

### 交付物
- Todo 列表轻量化方向说明
- 关键状态展示优先级
- 与现有 UI 的差异清单

---

## V2-6：根据实现结果补充边界验证

### 目标
在 V2 实现推进后，验证概念迁移没有破坏 V1 中仍需保留的业务能力。

### 至少关注以下场景
1. 临时事项仍不回写种草清单
2. 从种草加入 Todo 后，事项来源仍可追踪
3. 必要事项、重复事项、分次事项能力未被概念迁移破坏
4. 白天 / 晚上语境仍成立
5. 初始化页默认项与产品文案保持一致
6. 单页结构下，用户无需在双模式之间切换即可完成主路径

### 交付物
- V2 验证清单
- 手动回归记录
- 如有必要，再补充测试方案
