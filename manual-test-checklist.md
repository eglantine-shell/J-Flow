# V2 手动验证清单

当前版本目标：确认 J-Flow V2 已达到正式试用前的基础可用状态。

## 1. 初始化流程
- 清空本地数据后首次进入应用，应先看到初始化页。
- 页面标题应为：`哪怕你是一个100%的J人，想必也会有……`
- 默认种草清单应为：`阅读`、`观影`、`旅游`
- 默认有空就做应为：`工作日晚上`、`周末`、`长假`
- 两组 tag 都支持原位新增和删除。
- 每组至少保留一个 tag，删除到只剩一个时应被阻止。
- 输入空白内容时不能保存。
- 完成初始化后应进入主页。

## 2. 测试重置
- 进入设置页。
- “排序设置”应可正常切换正序 / 倒序。
- “重置应用（测试用）”应可用。
- 重置后应回到初始化页，并重新看到 V2 默认文案与默认项。

## 3. 普通 Todo 添加 / 完成 / 删除
- 在主页 Todo 顶部输入一条文本。
- 选择白天或晚上。
- 不展开“更多”，直接点击 `+`。
- 应创建一条普通临时 Todo。
- 新条目应可勾选完成。
- 新条目应可删除。
- 该条目不应回写到种草清单。

## 4. 必要 Todo 添加
- 在 Todo 添加区展开“更多”。
- 勾选“必要事项”。
- 创建一条 Todo。
- 新条目应展示必要事项状态。
- 该属性只影响本次实例，不应新增或改写种草条目。

## 5. 准备备注 Todo 添加
- 在“更多”里勾选“需要准备”。
- 输入准备备注。
- 创建 Todo。
- 新条目应展示准备状态与备注。
- 取消“需要准备”后，准备备注输入应隐藏且不应残留旧值。

## 6. 分次 Todo 添加与推进
- 在“更多”里勾选“分次事项”。
- 创建 Todo。
- 新条目应以分次事项实例写入。
- 应可执行分次推进。
- 进度更新后刷新页面，状态应保留。

## 7. 从种草添加到 Todo
- 在 Todo 添加区展开“更多”。
- 打开“从种草添加”。
- 推荐或候选列表应能打开。
- 选择一条种草后，应加入当前白天 / 晚上语境的 Todo。
- 同一日期同一时段重复添加同一条种草时，应提示不可重复添加。
- 页面上不应再有独立的拔草辅助区。

## 8. 种草新增 / 编辑 / 停用
- 展开主页下方“种草”区。
- 可新增一条种草。
- 可进入管理种草并编辑已有条目。
- 可停用一条种草。
- 停用后，该条目不应继续参与未来自动生成与拔草推荐。

## 9. tag 新增 / 删除
- 在种草新增或编辑表单中，可新增“种草清单”和“有空就做” tag。
- tag 删除应遵守“至少保留一个”的约束。
- 被使用中的种草清单删除时，应遵守当前限制并给出明确提示。

## 10. 日期切换后自动事项同步
- 在主页切换到其他日期，再切回今天。
- 自动事项应按当前既有规则同步显示。
- 白天 / 晚上分组应继续成立。
- 日期切换不应影响手动临时 Todo 与从种草添加 Todo 的实例语义。

## 11. 刷新页面后的基本状态
- 刷新后应仍停留在主页结构：`Todo + 种草`
- 不应重新出现“决策 / Todo”模式切换。
- 已保存的初始化结果、本地 Todo、种草和设置应保留。

## 12. 构建与 lint
- 执行 `pnpm run build`
- 执行 `pnpm run lint`
- 两项都应通过；若仅有既有 chunk size warning，可记录但不阻塞试用。

## 当前明确未覆盖能力
- 临时重复 Todo
- 拖动排序 / 拖动改变晚间语境
- 导出 / 导入
- 已归档种草恢复
- 搜索 / 筛选
- 自动化业务测试
- iOS App 迁移
