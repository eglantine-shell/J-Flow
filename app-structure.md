# 应用结构文档

本文档定义当前 V2 方向下的页面结构、主视图与用户路径。

说明：
- V1 已完成可试用 MVP，相关结构记录仍保留在历史文档与日志中
- 当前文档以 V2 目标结构为准
- 本轮只更新结构方向，不改现有实现

---

## 一、整体结构

V2 仍是单人、本地网页应用。

核心由以下部分组成：
1. 首次初始化页
2. 主页面
3. 设置相关页面 / 弹窗
4. 种草条目管理相关页面 / 弹窗

主页面仍是应用核心。

---

## 二、首次初始化页

### 目标
让用户快速建立两套基础认知：
- 自己平时会积累哪些“种草”
- 自己的“有空”通常长什么样

### 内容
1. 欢迎说明
2. 种草清单默认分类设置
3. 有空就做场景设置
4. 完成并进入应用

### 后续文案方向

标题：
- 哪怕你是一个100%的J人，想必也会有……

模块一：
- 种草清单
- 示例文案：比如，书单、影单、探店清单、逛公园清单，甚至……待处理的家务死角清单？
- 默认项目：阅读、观影、旅游

模块二：
- 有空就做
- 示例文案：当你将某些事定义为“有空就做”的事，期待的是怎样的“有空”？
- 默认项目：工作日晚上、周末、长假

### 交互要求
- 系统提供默认项
- 用户可新增
- 用户可删除
- 至少保留一个分类和一个场景后才能进入主页面

---

## 三、主页面

V2 主页面当前已迁移为四个主要区域：
1. 头部
2. 日期
3. Todo
4. 种草

这是 V2 与 V1 的核心结构差异。

V1 的“决策模式 / Todo 模式”双模式结构，已不再作为当前主表达。

---

## 四、头部

### 展示内容
- 应用名称
- 设置入口
- 必要的辅助状态入口

### 目标
- 保持轻量
- 不抢占 Todo 主体

---

## 五、日期区

### 展示内容
- 当前选中日期
- 星期信息
- 日期切换能力

### 目标
- 日期仍然是当天 Todo 的上下文锚点
- Todo 与拔草行为都围绕当前日期展开

---

## 六、Todo 区

### 页面目标
Todo 是主页面主体，用于承载某一天真正要做的事。

### 展示内容
- 临时事项
- 必要事项
- 重复事项实例
- 从种草中加入的事项

### 页面要求
- 默认录入足够轻量
- 允许快速添加临时事项
- 允许完成事项
- 允许查看准备信息
- 允许处理分次事项进度
- 继续保留白天 / 晚上语境

### 添加入口方向
Todo 添加入口当前已融合两类动作：
- 直接新增临时事项
- 从种草中选择 / 推荐拔草

默认行为应是：
- 先快速记一条事项

展开“更多”后可配置：
- 从种草中选择 / 推荐拔草
- 必要事项
- 需要准备
- 分次事项

当前暂不支持：
- 临时 Todo 的重复规则
- 若后续需要支持，需单独设计与模板层、自动生成逻辑的关系

---

## 七、种草区

### 页面目标
承载长期保留、等待合适时机再做的内容。

### 展示内容
- 种草条目列表
- 种草分类
- 适配场景
- 可进入 Todo 的入口

### 定位说明
- 种草区是独立清单
- 它不再承担普通待办职责
- 它的价值在于积累与后续拔草，而不是取代 Todo

---

## 八、拔草推荐

### 页面目标
当用户有空时，从种草区中帮助其挑选适合当前语境的事项。

### 表达方式
V2 中，拔草推荐不再需要单独塑造成“决策模式”。

更合适的结构是：
- 作为 Todo 添加入口中的一种动作
- 或作为种草区进入 Todo 的辅助入口

### 保留能力
- 仍可参考现有分类与场景结构
- 仍可保留白天 / 晚上语境

---

## 九、设置页 / 设置弹窗

V2 暂时仍保留：
- 分类管理
- 场景管理
- 排序或偏好设置
- 数据测试与重置入口

说明：
- 本轮不改这些结构
- 后续只需逐步把文案与产品表达对齐到 V2

---

## 十、种草管理页 / 详情弹窗

后续仍可保留种草条目管理能力，例如：
- 查看已有条目
- 编辑条目
- 停用条目
- 查看准备、重复、分次等属性

但产品表达上应从“模板管理 / 决策库管理”迁移为“种草管理”。

---

## 十一、V1 结构迁移说明

以下结构关系在 V2 中需要重写：

- V1 的模式切换区：后续应移除
- V1 的决策模式主区：后续应拆解并并入 Todo 区与种草区
- V1 的底部添加区：后续应改为更贴近 Todo 主流程的添加入口

以下结构继续保留价值：
- 初始化页
- 日期切换
- 设置页
- 白天 / 晚上语境

---

## 十二、当前不做

本轮只更新结构文档，不做：
- UI 实现
- 路由调整
- 页面重构
- 功能迁移
