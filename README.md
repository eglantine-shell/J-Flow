# J-Flow

J-Flow V2 的产品本体调整为：

普通 Todo List + 种草清单 + 有空时帮你拔草。

V1 试用后确认，真正有价值的不是“所有事务都进入决策库”，而是把日常待办、长期想做的事、以及“有空时做什么”的决策拆开处理：
- 普通日常事务优先走 Todo
- 长期想做、想看、想去、想处理的内容进入种草清单
- 当用户处于某种“有空”语境时，系统再帮助用户从种草清单里推荐可拔草项

## V2 核心概念

### 1. Todo 成为主体
- 主页面主体应回到 Todo
- 新增事项时，默认按临时事项 / 普通待办处理
- 用户不应为了记录一个简单待办，先经过模板、推荐、选入流程

### 2. 决策库改写为种草清单
- 原“决策库”在产品表达上改为“种草清单”
- 它不再承担普通待办职责
- 它更适合承载书单、影单、探店清单、旅行想法、想整理的角落等“先记住，等有空再做”的内容

### 3. 决策推荐改写为拔草推荐
- 原“决策推荐”在产品表达上改为“拔草推荐”
- 它不是安排全部生活的核心入口
- 它是 Todo 之外的增强能力：当用户有空时，帮用户从种草清单里挑可做项

### 4. 活动类型与时间场景的 V2 表达
- 底层数据结构暂时仍沿用 `activityTypes` 与 `sceneTags`
- 但在产品表达上：
  - `activityTypes` 更接近“种草分类”
  - `sceneTags` 更接近“有空就做场景”

## V2 页面结构

主页面当前结构为：
1. 头部
2. 日期
3. Todo
4. 种草

V2 不再继续强化“决策模式 / Todo 模式”双模式表达，Todo 已成为主页第一主体。

## 初始化页当前状态

初始化页当前已调整为：

### 哪怕你是一个100%的J人，想必也会有……

#### 种草清单
比如，书单、影单、探店清单、逛公园清单，甚至……待处理的家务死角清单？

默认项目调整为：
- 阅读
- 观影
- 旅游

#### 有空就做
当你将某些事定义为“有空就做”的事，期待的是怎样的“有空”？

默认项目调整为：
- 工作日晚上
- 周末
- 长假

## V2 已落地业务方向

当前已落实：

1. Todo 成为主页面主体
2. 种草成为独立清单，不再承担普通待办职责
3. 临时事项与从种草添加已在 Todo 入口中融合
4. 新增事项时默认为临时事项
5. 可以展开更多选项：
   - 从种草中选择 / 推荐拔草
   - 必要事项
   - 需要准备
   - 分次事项
6. 白天 / 晚上语境继续保留
7. V2 暂时尽量复用现有数据结构，除非后续实现中发现必须调整

- 已归档种草恢复
- 搜索 / 筛选

- `决策推荐`：重命名为“拔草推荐”
